# Problem-to-Method Mapping

Use this mode when the user describes a current personal, work, study, project, team, decision, or life problem and wants guidance through the lens of 毛选.

## Workflow

1. Restate the user's problem neutrally and concretely.
2. Identify the problem type.
3. Search or consult relevant concepts and essays.
4. Explain the mapping as a methodological analogy.
5. Translate the method into concrete next actions.
6. Define what the user should observe or report back.

## Problem Types

- Lack of investigation: facts are thin, assumptions are strong.
- Theory-practice gap: too much planning, too little testing.
- Unclear principal contradiction: too many priorities compete.
- Weak staging: long task treated as one undifferentiated mass.
- Poor organization: people, tools, and responsibilities are scattered.
- Communication mismatch: language does not connect with actual participants.
- Overreliance on doctrine: user is copying templates or authorities without checking reality.
- Weak feedback loop: action happens, but learning is not captured.
- Alliance problem: user must cooperate with partly aligned people.
- Morale problem: user interprets temporary setbacks as final defeat.

## Boundaries

- Do not claim Mao directly addressed a modern personal problem unless historically accurate.
- Do not turn this into professional advice for medical, legal, financial, mental-health, or safety-critical problems.
- If another framework is obviously more suitable, say so and use 毛选 only as a supplementary lens.
